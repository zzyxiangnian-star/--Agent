# 安全政策

## 支持范围

项目当前处于早期开发阶段，仅 `main` 分支的最新版本接受安全修复。

## 报告漏洞

请不要为安全漏洞创建公开 Issue，也不要在讨论区披露复现细节。

请在 GitHub 仓库的 **Security** 页面选择 **Report a vulnerability**，通过 Private Vulnerability Reporting 提交：

- 受影响的功能或版本；
- 漏洞影响与风险；
- 可复现的最小步骤；
- 已知的缓解或修复建议。

维护者会尽快确认报告、评估影响，并在修复发布前协调披露时间。若 GitHub 私密报告入口暂时不可用，请等待入口恢复，不要公开漏洞细节。
